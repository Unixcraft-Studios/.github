# Welcome to Unix-Craft Studios.
 **Two programmers, opposite sides of the planet, one philosophy: UNIX.**

## About

Unixcraft Studios is a collaborative software collective dedicated to solving modern computing problems through the principles
of the UNIX philosophy and suckless design paradigm. We believe that software should be simple, modular, and fast, crafted with
 intention, not bloat.

From low-level systems tooling to high-performance web applications, from kernel-level optimizations to custom rendering engine
s, we approach every project with the same core tenets: do one thing and do it well.

## Philosophy

We operate under the UNIX and KISS/Suckless principles:

- **Simplicity First**: Clean, readable code over clever abstractions
- **Do One Thing Well**: Focused tools that compose elegantly
- **Fast by Default**: Native performance without compromise
- **Transparency**: Code that you can understand, audit, and modify
- **Minimal Dependencies**: Reduce complexity at every layer

## What We Build

### Systems Tooling
Lightweight, efficient command-line utilities designed to integrate seamlessly into UNIX pipelines.

### Web Applications
High-performance web applications that respect both the end-user and the developer, built without unnecessary frameworks or run
time overhead.

### Kernel Patches & Optimizations
Targeted kernel modifications that improve performance, reduce latency, and maintain system transparency.

### Custom Rendering & UI
Leveraging **raylib** and **custom GLSL shaders** to create native, GPU-accelerated interfaces that run fast without the bloat
of traditional UI frameworks. Every pixel counts.

## Architecture & Design

All Unixcraft projects follow these architectural principles:

*UNIX Philosophy → Clean Abstractions → Modular Components → Native Performance*

- **Modular Design**: Each component has a single responsibility
- **Standard Interfaces**: POSIX compliance where applicable, plain-text where appropriate
- **Minimal Overhead**: Direct hardware access, no unnecessary virtualizations
- **Composability**: Tools and libraries that work together, not against each other
- **Performance-First**: Profiling and optimization are not afterthoughts
